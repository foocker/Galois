# Writing Standards

Use this file before delivering review text or writing final artifacts.

## Evidence Discipline

A strong claim such as "the method is flawed", "the theorem is not proved", "the contribution is not novel", or "the comparison is insufficient" requires manuscript evidence, source evidence, inspected data, or a concrete derivation.

Interpretive claims require textual support. If the evidence is incomplete, present the item as a question, inspection limit, or `personal inference`.

## Prohibited Expressions

Avoid inflated language: "breakthrough", "revolutionary", "paradigm-shifting", "powerful capability", "fully autonomous", "shockingly", "obviously", and "needless to say."

Avoid generic filler: "it is worth noting", "in conclusion, this paper", excessive bullet fragmentation, and ornamental praise.

Do not expose internal traces in delivered output: file paths, skill names, internal rule labels, sub-agent details, orchestration details, or inline labels like `Claim`, `Evidence`, and `Warrant`.

## Structure

- Lead with the verdict.
- Use coherent paragraphs for arguments.
- Use bullets only for parallel items or task lists.
- Keep one main claim per paragraph.
- Make transitions carry argument, not procedure.
- Localize every comment and make it actionable.

## Delivery Hygiene

- Markdown tables are fine in files; avoid them in chat unless the user asked for a table.
- Long reports should be delivered as artifacts with a short summary.
- Use ASCII filenames for attachments and generated artifacts.
- Do not copy long passages from papers or books.

## Pre-Delivery Self-Check

Before delivery, verify:

- Every substantive claim has evidence or is labeled `personal inference`.
- Absolute assertions have scoped precedent checks or are softened.
- Strong claim phrases in the manuscript have been checked against technical content.
- Positive characterizations pass the uniqueness test.
- Contribution judgments are benchmarked against a reference class.
- Direct-predecessor claims note whether full text was inspected.
- Figure/table critiques are based on actual inspection or marked as unverified.
- Proof review depth is labeled.
- No internal process labels, file paths, or harness traces appear in delivered text.
- Revision tasks are concrete and inspectable.
- Paper writing page artifact outputs match the requested mode and artifact contract.

If any item fails, revise before delivery.
